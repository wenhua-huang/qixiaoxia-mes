## 一、客户概况

**圣享实业** — 纸袋制造工厂，主要产品为各类纸袋（奔趣系列/SOS方底袋/PICO纸袋/王冠纸袋/黑色旗帜纸袋/单杯袋/边框纸袋等），业务分为外贸（陈仁林）、内贸（陈丽丽）、现货三条线。

### 组织架构与角色

| 角色 | 姓名 | 职责范围 | 核心表单 |
|------|------|---------|---------|
| 仓库主管 | 姚晓翔 | 原料/成品库存、出入库、发货 | 月度库存表（原料+成品3Sheet）、出入库单、出货清单 |
| 工厂主管 | 吴兵 | 生产计划、工序排产、辅料库存、设备 | 生产总单(20Sheet)、流转单、辅料出入库 |
| 生产外发主管 | 鞠阳 | 外发采购、物料采购、生产流转 | 采购单(多家供应商)、外发加工单 |
| 行政/财务 | 陈慧琳 | 供应商对账、考勤管理 | 对账表、加班单、考勤表 |

### ⭐ 外发加工模式（核心发现）

**100% 带料外发**：圣享采购所有主要原材料 → 发给外协厂加工 → 外协厂只收加工费。

#### 工序分配规则

```
订单 → 印刷+制袋（整包，不拆分）：
    ├── 自制：圣享 1/2号印刷机 + 1~9号制袋机
    └── 外发：万隆 / 吉荣 / 浩卓 / 欧诺（整单外发，一家全包）

订单 → 贴绳（独立工序，统一外发）：
    └── 外发：温州圣皓包装（专业贴绳合作伙伴）
```

- **印刷+制袋不拆分工序**：不存在"印刷给A厂、制袋给B厂"的情况，整包给一家外协厂
- **贴绳是独立外发工序**：半成品袋子+绳子→发给圣皓→贴绳完成→返回

#### 证据链

1. `心心纸袋-小号贴绳加工单.docx`：明确写明"来料数量 98,000个"、"红色圆纸绳，**圣享提供**"
2. `外发生产计划` Sheet：备注列标注"万隆生产"/"吉荣生产"/"浩卓生产"——每单只指定一个外协厂
3. `外发生产计划` 和 `外发贴绳计划` 订单号完全不重叠——不同订单在不同阶段

**关键需求**："一单到底"的流转单聚合视图，将工单-路线-报工-出货信息整合在一个页面。

## 四、核心差异化需求（纸袋行业 vs 通用 MES）

以下需求是圣享工厂特有的，ktg-mes 没有或需要大改：

### 4.1 纸张双单位管理

纸张行业特征：同一物料需要同时追踪 **卷数**、**重量(吨)**、**米数** 三个维度。

- ktg-mes：`md_item` 有 `unit_of_measure` 单字段，不支持双单位
- qixiaoxia-mes 需实现：
  - `qxx_md_item` 增加 `unit2`（辅助单位）和 `conversion_rate`（换算率，如 1卷 ≈ 0.697吨）
  - 所有库存交易同时记录两个单位的变动
  - 月度汇总视图同时显示卷数和重量

### 4.2 纸张分切/复卷工序

客户有"印刷车间 纸张分切"工序：一卷宽幅纸 → 分切成多个窄幅 → 产生纸边（辅料入库）。

- ktg-mes：无此工序类型
- qixiaoxia-mes 需实现：
  - 工艺路线中增加"分切"工序类型
  - 分切后动态生成子批次（一个母卷→多个子卷）
  - 分切损耗（纸边）自动生成辅料入库记录

### 4.3 生产流转单（一单到底）

- ktg-mes：工单/路线/报工/入库分离
- qixiaoxia-mes 需实现：
  - "流转单"聚合视图：订单信息 + 原料纸 + 印刷工序 + 制袋工序 + 包装 + 发货
  - PC端一屏展示完整流转信息
  - 移动端扫码流转

### 4.4 外发工序报工（两种模式）

客户要求：外发进度支持报工，可由内部员工代填或供应商直填。

- ktg-mes：外协物料移动（issue→recpt），无工序级别报工
- qixiaoxia-mes 需实现：
  - 外发工序纳入工艺路线，作为普通工序节点（标记为外发）
  - **模式A**：内部跟单员代填（复用 pro_feedback，填报人=内部员工）
  - **模式B**：供应商直填（供应商账号+权限隔离+自助报工页面）
  - 外发报工关联 `qxx_wm_outsource_issue`（哪批料发出的）

### 4.5 机台排产总表

- ktg-mes：甘特图排产，无表格化的机台维度汇总
- qixiaoxia-mes 需实现：
  - 按设备（印刷机/制袋机）的排产表格视图
  - "执行中总单"：工序进度(0/11)、工单进度、工期状态一目了然
  - 外发任务作为一个虚拟"机台"纳入排产

### 4.6 月度库存汇总视图

客户每月的核心报表，ktg-mes 标准功能不包含。

- 月初库存 + 本月入库/出库 + 剩余库存 + 每日明细（1~31号）
- 原料（按材料+门幅）、成品（按外贸/内贸/现货分类）
- 辅料（按辅料类型，含预警状态）

## 八、风险与建议

### 风险

1. **工作量巨大**：从空白脚手架到圣享工厂完整MES，18~24周（1人全职），需合理控制范围
2. **采购模块从零构建**：ktg-mes 无采购订单参考，需独立设计
3. **供应商门户安全**：外部用户访问系统，需严格的权限隔离和数据安全
4. **数据迁移**：客户 Excel 数据命名不规范、日期格式不统一、有公式错误，导入前需清洗

### 建议

1. **MVP 优先**：先上线 Phase 1（MD+WM），让仓库先用起来，边用边迭代
2. **外发报工先做代填模式**：Phase 2 先实现内部员工代填，供应商门户放到 Phase 3
3. **采购模块可后置**：先允许无PO入库（杂项入库），采购模块放到 Phase 4
4. **报表用 UReport 渐进开发**：不追求一次性覆盖所有报表，按需开发
