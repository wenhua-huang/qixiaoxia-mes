# 测试约定 (Testing Conventions)

## Test Hierarchy (测试金字塔)

| 层级 | 位置 | 框架 | 运行速度 | 范围 | 真实接口调用 |
|------|------|------|---------|------|:---:|
| Unit | `backend/*/src/test/` | JUnit 5 + Mockito | 秒级 | 单个 Service/Utils 方法 | ❌ **禁止** |
| Integration | `backend/ruoyi-admin/src/test/` | SpringBootTest + Testcontainers | 分钟级 | Controller → DB 全链路 | ✅ 真实 DB |
| Component | `frontend/src/**/__tests__/` | Vitest + @vue/test-utils | 秒级 | 单个 Vue 组件 | ❌ **禁止** |
| E2E | `e2e/tests/` | Playwright + TypeScript | 分钟级 | 全栈业务流程 | ✅ 真实前后端 |

## 接口调用禁令

**单元测试和组件测试严禁发起真实网络请求，违者必须重写。**

| 层级 | 禁止行为 | 必须做法 | 违规示例 |
|------|---------|---------|---------|
| **后端单元测试** | 连数据库、调 Redis、发 HTTP 请求 | Mock 所有 Mapper/Service/外部依赖 | `@Autowired Mapper` 直接查库 |
| **前端组件测试** | 调真实后端 API（`axios.get('/api/xxx')`） | `vi.mock('@/api/xxx')` mock 所有 API 模块，返回假数据 | 组件 mount 后触发 `getList()` 发了真实请求 |
| **集成测试** | 连生产库 `mes` | Testcontainers 临时 MySQL，`mes_integration_test` 库 | `spring.datasource.url` 指向 `localhost:3306/mes` |
| **E2E 测试** | Mock API 响应 | ⚠️ **必须连真实前后端**，`page.route()` 仅用于模拟不可控的外部服务 | `page.route('/api/**', ...)` 拦截业务 API |

> **判断标准**：断网后能否跑通？单元测试和组件测试必须能，集成测试和 E2E 必须不能。

## Database Rules for Tests

1. **单元测试**不连接数据库（Mock 所有 Mapper）
2. **集成测试**使用 Testcontainers MySQL 8.0.33，不连本地/开发数据库
3. 测试数据库名：`mes_integration_test`，字符集 utf8mb4
4. Schema 通过 Spring `sql.init` 从 classpath `sql/ry_20260417.sql` 自动初始化（源文件在 `backend/sql/`，Maven 资源插件拷贝到 `test-classes`）
5. 每个测试类在 `@BeforeEach` 中清理目标表
6. 生产库 `mes`，测试库 `mes_integration_test`，严禁交叉

## Naming & Directory Conventions

**Backend**:
- Unit test: `src/test/java/{包路径}/{类名}Test.java` → 由 Surefire 执行 (`mvn test`)
- Integration test: `src/test/java/{包路径}/{类名}IT.java` 或 `{类名}IntegrationTest.java` → 由 Failsafe 执行 (`mvn verify`)
- Method naming: `should_{expected}_when_{condition}` (snake_case 英文)
- `@DisplayName` 中文描述

**Frontend**:
- Test files: `__tests__/{组件名}.spec.ts`（与组件同目录）
- `data-testid` 属性用于元素定位（优先于 CSS class）

**E2E**:
- Files: `e2e/tests/{模块}/{用例}.spec.ts`
- Selector 优先级: `data-testid` > `aria-label` > CSS class

## Runtime Conventions

**本地开发验证**（提交前手动执行）：
```bash
cd backend && mvn test           # 单元测试
cd frontend && npm test           # 组件测试
```

**全量验证**（合并前手动执行）：
```bash
cd backend && mvn verify          # 单元 + 集成测试（需 Docker）
cd frontend && npm test           # 组件测试
cd e2e && npx playwright test     # E2E 测试（需前后端都在运行）
```

> ⚠️ 当前项目**未配置 CI/CD**（无 GitHub Actions / GitLab CI / Jenkins），上述命令需手动执行。Pre-commit hooks（husky）也尚未配置。

## Prompt Hints for Claude

写测试时遵循以下模式（**单元/组件测试严禁真实网络请求**）：

**Backend unit test**: `@ExtendWith(MockitoExtension.class)`, Mock **所有**依赖（含 Mapper、Service、Redis），不启动 Spring 容器，覆盖正常+边界+异常
**Backend integration test**: 继承 `BaseIntegrationTest`, 用 `RestTemplate`, DB 断言。**仅此层可连真实 DB（Testcontainers）**
**Frontend component test**: **必须** `vi.mock('@/api/xxx')` mock 所有 API 模块，mock Router/Pinia，用 `data-testid`。**禁止**在组件测试中发真实 HTTP 请求
**E2E test**: 复用 `storageState`, `waitForResponse` 等 API 返回, 不用 `sleep`。**必须**连真实前后端，**禁止** mock 业务 API
